## 1. 재반등 레이더 조건 개편 (`573ff20`, `7d92d52`)

기존 "급등 후 식은 종목(fade)" 위주에서 **재매집·재반등(reaccum)** 감시로 확장.

- **모집단(reaccum)**: 13일 내 폭발(거래대금 1천억+ · 고가 +13%) → 식음 → MA20 위 → 투신(ivtr) 매집 → **재반등 10분봉**.
- **재반등 게이트 종가 기준 완화**: 고가 기준 → **종가 등락률 기준**(`REACCUM_CHANGE_MIN=-4.0`, `MAX=10.0`)으로 변경. (HL만도 같은 식은 종목이 누락되던 문제 해결)
- **재반등 봉 탐지**: `reignition_bars(bars, body_pct_min, value_min)` — 몸통 2%+ · 거래대금 30억+ 조건을 만족하는 **모든 10분봉**을 반환. 스캔은 그중 몸통 최대 봉을 대표로 쓰고 `reignition_bars` 필드로 전체를 실음.
- `score_raw=0` + `visible_experimental=True`로 격리 — 화면엔 보이되 **core 통계·가중치 튜닝엔 미반영**(드리프트 방지).

## 3. 변별 점수 (표시 전용 강도) (`1a9891f`)

재매집 후보가 전부 62점으로 동점이라 정렬이 무의미하던 문제 해결.

- `breakdown` = base 62 + `re_value`(0~12) + `re_body`(0~6) + `re_count`(0~6) + `flow`(0~8) + `explosion`(0~6), `score = min(95, 합)`.
- **표시·정렬 전용** — `score_raw=0` 유지로 통계엔 영향 없음.

## 4. 3일 내 +7% 과거확률 라벨 (forecast) (`0641b31`)

"오늘 사면 내일 +7%"는 실측상 불가능(내일 +7% 터치 26%) → **확률 라벨**로 정직하게 재정의.

- 6개월 전종목 백테스트 기반 forecast 모델(z-sum: mom3·mom5·MA20 이격·거래량 급증).
- 후보별 `forecast` 필드 부여 → SuspectCard에 **"유사셋업 3일내 ~N%(강 모멘텀) · 내일1일 +7%는 ~M% · 코호트 통계·보장 아님"** 표기.
- 거래량 급증(vsurge)은 **유효 거래일 10일 이상일 때만** 계산(희소·0값 인플레 방지).

## 5. 분할 전략 실측 트래커 (`cc93975`)

레이더 신호를 실제 매매했다 가정한 **실현 net 성적을 라이브 누적**.

- 가정: **20/30/50 분할 매수 + 익절 +7% / 손절 −5%(종가)**, forward 10거래일 일봉, **수수료 0.3%p 차감**.
- `radar_backtest.py`: `_strategy_outcome` / `strategy_eval`(멱등 · age 16일 게이트 · 40일 만료) / `strategy_sim_stats` → `performance.json`의 `strategy_sim` 블록.
- `/performance` **StrategySimPanel**: 거래수 · 승률 · net 평균 · 손절률 · 수익거래% · 최악. `min_n=30` 미만은 "수집 중". **표시 전용 · core 통계 미반영 · 보장 아님 · 종가손절 가정** 명시.
- 검증: 단위테스트(win/stop/dip/부족) + 집계 PASS · 웹빌드 통과 · **적대적 코드리뷰 2R 수렴**(P1/P2 0).
