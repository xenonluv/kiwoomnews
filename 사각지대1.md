## 1. 결론 요약

SK이터닉스가 2026-07-22 레이더의 final suspects에 나오지 않은 원인은 데이터 누락이나 API 장애가 아니다. 종목은 장중 스캔에 계속 관찰됐고 `youtong_current`에도 들어왔지만, 아래 두 개의 기존 진입 경로 사이에서 탈락했다.

1. 폭발형 경로: 회전율이 `69.4%`로 최소 기준 `90%`에 미달했다.
2. 흔들기 경로: 고점 대비 밀림이 `13.96%p`로 최소 기준 `15%p`에 `1.04%p` 부족했다.

당일 수치는 고가 상승률 `+24.87%`, 종가 상승률 `+10.91%`, 회전율 `69.4%`였다. 즉, 고가가 크게 상승한 뒤 종가도 상당 부분을 지킨 `강한 종가 유지형/연속 상승형`이었으나 현재 분류에는 이 유형을 담는 별도 통로가 없다.

다음 거래일인 2026-07-23 09:53 KST 로컬 스캔에서는 전일 종가 62,000원 대비 고가 75,000원, `+20.97%`를 기록했다. 프로그램의 핵심 KPI인 익일 고가 `+7%`와 `+13%`를 모두 달성했으므로, 이번 사례는 결과적으로 의미 있는 false negative다.

다만 한 건의 성공 사례만으로 기존 흔들기 기준을 낮춰서는 안 된다. 권고안은 기존 suspects와 자동매매를 그대로 둔 채, 경계값 바로 아래의 종목을 별도 `관찰용 shadow cohort`로 저장하고 forward 성과를 쌓는 것이다.

## 2. 분석 시 지켜야 할 기준

이 프로젝트의 성과 기준은 익일 종가가 아니라 신호일 종가 대비 다음 거래일 장중 고가다.

- 최우선 KPI: 익일 고가 `+7%`, `+13%` 도달 여부
- 보조 KPI: `+11%`, `+15%`, `-5%` 도달 여부
- 매매 순서 KPI: 분봉 기준 `+7% 선행`, `-5% 선행`, 같은 분봉 동시 도달 여부
- 종가수익은 이번 사각지대 판정의 성공·실패 기준으로 사용하지 않는다.

또한 다음 세 통계를 섞지 않는다.

- `signal_time_prior`: 신호 당시 저장된 사전 통계
- `current_retro_cell`: 현재 규칙으로 과거를 재분류한 소급 통계
- `current_forward_cell`: 신호 당시 저장된 조건의 실제 익일 결과

이번 SK이터닉스 사례는 2026-07-22 당시 저장된 원시 스캔과 2026-07-23 실제 시세를 연결한 사례다. 향후 새 후보군의 성능 판단은 반드시 forward 표본으로 해야 한다.

## 5. 경계값 관점에서 본 사각지대

### 5.1 흔들기 기준과의 거리

- 필요 밀림: `15.00%p`
- 실제 밀림: `13.96%p`
- 부족분: `1.04%p`

고가 상승률 `24.87%`가 그대로였다고 가정하면, 종가 상승률이 `+9.87%` 이하일 때 기존 흔들기 조건을 통과한다. 가격으로는 대략 61,400원 이하이며 실제 종가 62,000원과 약 600원, 약 `0.97%` 차이다.

즉, 종가가 약 1%만 더 낮았으면 흔들기로 분류될 수 있었지만 종가를 더 강하게 지켰다는 이유로 오히려 final suspects 후보가 되지 못했다. 이는 현재 필터가 의도한 흔들기 탐지에는 맞지만, 익일 연속 상승 후보 탐지라는 전체 목적에서는 사각지대가 될 수 있다.

### 5.2 폭발형 기준과의 거리

- 필요 회전율: `90.0%`
- 실제 회전율: `69.4%`
- 부족분: `20.6%p`

폭발형 기준과는 거리가 크다. 따라서 이 사례를 해결하기 위해 폭발형 회전율을 90%에서 약 69%까지 낮추는 것은 영향 범위가 너무 크다. 다수의 일반 급등주가 새로 유입되어 기존 폭발형 의미와 통계가 오염될 가능성이 있다.

### 5.3 현재 분류 구조상 빈 구간

SK이터닉스는 다음 세 분류 어디에도 정확히 속하지 않았다.

- 폭발형: 고가는 충분하지만 회전율 90% 미만
- 흔들기: 고가와 회전율은 충분하지만 밀림 15% 미만
- 저점매집/reaccum: 종가 상승률 `+10.91%`로 기존 상단 범위를 벗어남

반면 youtong 조건에는 들어간다. 그러나 youtong은 유통 회전 관심종목을 보여주는 관찰·알림 경로이고, final suspects와 익일 고가 평가 모집단을 자동으로 구성하는 경로가 아니다. 따라서 `youtong에는 포착됐지만 모델의 forward 연구 표본으로 남지 않는` 구조적 간극이 생긴다.

## 7. 근본 원인

### 7.1 이산적인 하드 컷

현재는 `fade_pct = 14.99`면 완전 탈락하고 `15.00`이면 통과한다. 경계 바로 아래 사례를 별도로 저장하지 않으므로, 기준의 민감도와 경계 구간의 실제 성과를 forward 방식으로 검증하기 어렵다.

### 7.2 흔들기와 강한 종가 유지형의 혼동 가능성

고가에서 크게 밀려 종가가 낮아지는 전형적 흔들기와, 고가 대비 어느 정도 밀렸지만 종가 상승분을 많이 지키는 연속 상승형은 다른 패턴일 수 있다. 기존 `SHAKEOUT_FADE_PCT`를 낮추면 두 패턴을 한 통계에 섞게 된다.

### 7.3 youtong 관찰과 final 평가의 단절

youtong이 종목을 잡아도 final suspects가 아니면 일자별 공식 성과 분모에서 제외된다. 이는 기존 KPI 정합성 측면에서는 올바르지만, youtong 중 어떤 유형이 반복적으로 익일 급등하는지 연구할 별도 고정 표본이 부족해질 수 있다.

### 7.4 조기 `continue` 이후 정보 활용 불가

흔들기 밀림 기준에서 탈락한 뒤에는 정식 후보 생성·후속 평가 경로까지 도달하지 못한다. 원시 audit로 사후 원인 추적은 가능하지만, 신호 당시 조건을 불변 스냅샷으로 묶어 익일 평가하는 연구 cohort가 자동 생성되지 않는다.

### 7.5 결과를 본 뒤 기준을 맞추는 과최적화 위험

이번 한 종목을 잡기 위해 `15% → 13.9%`로 낮추면 SK이터닉스에는 맞지만 다른 날의 false positive가 얼마나 증가하는지 알 수 없다. 그러므로 현재 결과를 근거로 본 경계값을 즉시 운영 규칙으로 바꾸는 것은 금지해야 한다.

## 8. 최소 안전 개선안

### 8.1 기본 원칙

기존 흔들기, 저점매집, 폭발형, final suspects, ranking, autotrade는 그대로 둔다. 새 조건은 처음부터 운영 후보로 승격하지 말고 `관찰 전용 shadow cohort`로만 저장한다.

가칭:

- 내부 키: `strong_close_near_shakeout`
- 사용자용 표현: `강한종가형 관찰` 또는 `흔들기 근접 관찰`
- 규칙 버전: `strong-close-shadow-v1`

명칭은 Claude가 기존 스키마와 충돌 여부를 확인한 뒤 최종 제안하되, 기존 `강한흔들기`와 혼동되는 이름은 피한다.

### 8.2 1차 연구 조건안

다음은 운영 확정 기준이 아니라 forward 데이터를 모으기 위한 좁은 연구 가설이다.

```text
high_pct >= 20.0
turnover_pct >= 40.0
12.0 <= fade_pct < 15.0
7.0 <= change_pct <= 15.5
가격·고가·유통주식수 데이터 유효
당일 신호시각과 거래일 일치
```

조건 선택 이유:

- `high_pct >= 20`: 기존 흔들기와 같은 장중 상승 강도를 유지한다.
- `turnover_pct >= 40`: 기존 흔들기 최소 회전율을 유지한다.
- `12 <= fade < 15`: 기존 흔들기의 바로 아래 좁은 경계만 관찰한다.
- `change_pct >= 7`: 종가까지 상승 탄력을 유지한 종목으로 범위를 제한한다.
- `change_pct <= 15.5`: 기존 사전 필터의 상단 논리와 정합성을 유지하고 극단적인 상한가 근접 종목의 혼입을 제한한다.

SK이터닉스는 `high 24.87 / turnover 69.4 / fade 13.96 / change 10.91`로 이 연구 조건을 만족한다.

주의: 이 숫자들은 SK이터닉스 한 건에 맞춘 최종 최적값이 아니다. 후보군을 너무 넓히지 않으면서 경계 민감도를 검증하기 위한 첫 관찰 범위다. 실제 도입 전 Claude는 현재 코드의 정확한 사전 필터 상·하한과 단위, 결측치 처리를 다시 확인해야 한다.

### 8.3 저장 위치와 데이터 분리

단순히 `rank_policy.py`에 shadow bucket을 추가해서는 안 된다. SK이터닉스는 흔들기 gate의 조기 `continue`에서 이미 제거되므로 rank 단계에 도달하지 않는다.

따라서 스캔 단계에서 다음 중 하나의 최소 구조가 필요하다.

1. 기존 흔들기 탈락 직전에 연구 조건을 평가하고 별도 `shadow_candidates` 배열에 불변 스냅샷을 저장한다.
2. 또는 기존 후보 객체와 호환되는 관찰 객체를 만들되 `published=false`, `final=false`, `eligible_for_ranking=false`, `eligible_for_autotrade=false`를 명시하여 별도 경로로 내보낸다.

권장 저장 내용:

```json
{
  "code": "475150",
  "name": "SK이터닉스",
  "signal_date": "20260722",
  "signal_time": "205600",
  "entry": 62000,
  "pattern": "strong_close_near_shakeout",
  "rule_version": "strong-close-shadow-v1",
  "high_pct": 24.87,
  "change_pct": 10.91,
  "fade_pct": 13.96,
  "turnover_pct": 69.4,
  "main_shakeout": false,
  "published": false,
  "final": false,

  "material_snapshot": {},
  "risk_flags": [],
  "source_universes": ["naver_up", "shakeout_extra", "youtong_current"]

```

실제 키 이름은 기존 JSON 관례에 맞춰 조정할 수 있다. 중요한 것은 아래의 논리적 분리다.

- 기존 `data/radar_history/YYYYMMDD.json`의 final suspects 분모에 포함하지 않는다.
- 기존 bucket 통계에 합치지 않는다.
- 기존 shakeout 플래그를 `true`로 만들지 않는다.
- 자동매매 대상이 되지 않는다.
- 신호 당시 entry, 기술 수치, 재료 스냅샷, 위험 플래그를 이후 수정하지 않는다.
- 같은 종목이 여러 스캔에서 잡혀도 일자별 평가 단위는 중복되지 않도록 고유키를 정한다.

저장 파일은 기존 로컬 데이터 구조를 먼저 확인한 뒤 가장 작은 변경으로 결정한다. 예시는 다음과 같다.

```text
data/local/radar_raw/YYYY/MM/DD/research/strong_close_shadow.json
```

또는 기존 scan/history 문서의 하위 `shadow_candidates` 필드로 저장할 수 있다. 단, 기존 소비자가 알 수 없는 필드를 무시하는지, JSON 용량과 write 원자성에 문제가 없는지를 먼저 테스트해야 한다.

### 8.4 재료 정보도 신호시각 기준으로 함께 저장

이 cohort의 기술조건만 저장하면 나중에 재료 효과를 분석할 수 없다. 후보 생성 당시 아래 항목이 이미 존재하면 함께 스냅샷으로 고정한다.

- 재료등급 `S/A/B/C/D/N`
- 직접재료/간접재료
- 공시/뉴스 출처와 확인시각
- 재료 신선도와 발표 후 경과일
- 장중 신규 재료인지 기존 재료 재순환인지
- 희석, 권리, 투자주의·경고·거래정지 가능성 등 `risk_flags`

재료가 없거나 조회에 실패한 경우 임의로 등급을 추정하지 말고 `N` 또는 `unknown`으로 명시한다. 현재 뉴스로 과거 신호일 재료를 다시 칠한 값은 forward 통계에 사용하지 않는다.

### 8.5 화면 표시는 선택 사항

최소 패치의 핵심은 연구용 저장과 익일 평가다. UI 표시는 데이터 구조가 안정된 뒤 별도 승인으로 진행해도 된다.

만약 UI까지 포함한다면:

- final suspects에 섞지 않는다.
- youtong 카드나 별도 관찰영역에 작은 `강한종가형 관찰` 배지를 붙인다.
- 기존 `강한흔들기/약한흔들기` 배지로 표시하지 않는다.
- 추천, 매수 신호, 자동매수 가능이라는 오해가 없도록 툴팁에 `연구 표본·통계 미확정`을 명시한다.

## 9. 익일 평가 설계

shadow cohort도 기존 프로그램 목적과 동일하게 다음 거래일 고가로 평가해야 한다.

필수 결과 필드:

- `entry`: 신호일 KRX 종가
- 결과 거래일
- 익일 시가/고가/저가/종가
- `high_return_pct`, `low_return_pct`
- `hit_plus7`, `hit_plus11`, `hit_plus13`, `hit_plus15`
- `hit_minus5`
- 분봉 기준 `plus7_time`, `minus5_time`
- `first_touch`: `plus7_first`, `minus5_first`, `same_minute_unknown`, `neither`
- 장중 최고가 발생시각
- 평가 데이터 출처와 평가 완료시각

평가 시 주의사항:

- 다음 달력일이 아니라 다음 거래일을 사용한다.
- KRX `market="J"` 가격을 사용한다.
- 일봉에서 +7과 -5가 둘 다 확인되면 분봉으로 순서를 판정한다.
- 같은 1분봉 안에서 양 선을 모두 건드려 순서를 모르면 `same_minute_unknown`으로 남긴다.
- 기존 final suspects 성과와 shadow 성과를 별도 표로 집계한다.
- 동일 종목 반복 신호 수와 고유 종목 수를 모두 표시한다.

## 10. 승격 판단 기준

shadow cohort가 쌓였다고 즉시 suspects로 승격하지 않는다. 최소한 다음 조건을 검토한다.

- forward 표본 `n >= 20`
- 고유 종목 수 `>= 15`
- `+7%`, `+13%` 적중률을 각각 분자/분모와 함께 제시
- `-5%` 도달률 및 `+7/-5` 최초도달 성과 확인
- 기존 shakeout, 같은 기간 youtong-only 집단과 비교
- 월별 또는 최근 국면별로 효과가 한 시기에만 집중됐는지 확인
- 재료등급·직접성·신선도별 성과 분리
- 소급 표본과 forward 표본 분리
- 가능하면 Wilson 신뢰구간 하한까지 비교

승격 검토 시에도 다음은 별도 사용자 승인 사항이다.

- final suspects 포함
- 기존 bucket 변경 또는 신규 정식 bucket 추가
- ranking 가산점/하드 게이트 적용
- 자동매매 조건 적용

## 17. 최종 판단

SK이터닉스는 어제 레이더가 전혀 보지 못한 종목이 아니라, `폭발형에는 회전율 부족`, `흔들기에는 밀림 1.04%p 부족`, `저점매집에는 종가 상승률 과다`로 세 분류 사이에 걸린 종목이다. 다음 날 실제로 `+20.97%` 고가를 기록했으므로 이 경계 구간을 연구할 가치는 충분하다.

그러나 현재 증거는 한 건의 강한 사례다. 가장 안전하고 합리적인 수정은 기존 흔들기 문턱을 낮추는 것이 아니라 `강한 종가 유지형 근접 후보`를 별도 shadow로 저장하여, 기존 KPI와 동일한 방식으로 forward 성과를 검증하는 것이다. 이 방식이면 기존 레이더의 의미와 자동매매 안전성을 건드리지 않으면서, 실제로 반복되는 사각지대인지 확인할 수 있다.
