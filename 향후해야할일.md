# 향후 해야 할 일 — 예측 정확도 개선 로드맵 (백테스트 ~2주 누적 후)

> 목표: 백테스트로 쌓인 데이터를 근거로 **가중치·필터를 개선**하고, 나아가 **자동 반복 튜닝(adaptive)** 으로 정확도를 점진 향상.
> 핵심 원칙: **데이터(백테스트) 없이는 튜닝 금지**(과적합 방지). 숫자(calibration)는 이미 자동으로 정직해지지만, **예측력 자체는 이 작업으로만 오른다.**

---

## 0. 시작 조건 (먼저 확인) — Mac에서
```bash
cat analyzer/state/backtest.json     # sample ≥ 30~50, recent_hit_rate, avg_return
cat analyzer/state/calibration.json  # bins 채워졌는지(구간별 실제 적중률)
ls analyzer/state/history/           # 거래일 수 = 누적 일수
```
- **sample이 30 미만이면 아직 튜닝하지 말 것** (노이즈에 과적합). 더 누적 대기.

## 1. 데이터 진단 (튜닝 전 분석)
backtest 이력으로 다음을 본다(필요 시 `analyzer/backtest.py`에 분해 통계 추가):
- **지표별 적중 기여(lift)**: 각 강세 플래그(마감강도≥0.7 / 정배열 / MACD골든 / RSI강세 / Stoch골든 / 일목 구름위 / 거래량2배↑ / 호재 / 장중지속)가 **true일 때 vs false일 때** 적중률 차이.
- **국면별**: 저점/눌림목/상승추세/과다상승 각각의 실제 적중률.
- **확신등급별**: confidence 상/중/하 실제 적중률(상이 정말 더 맞는가).
- **점수구간별**: calibration bins(이미 산출). 단조 증가하는가?
→ "잘 맞는 신호"와 "헛신호"를 데이터로 식별.

## 2. 가중치·필터 튜닝 (1차, 수동·근거기반)
대상 파일: **`analyzer/run.py`의 `score()`** + `PHASE_ADJ` + 임계값, **`scripts/screener.py`의 필터 임계값**.
- 적중 기여 큰 지표 → **가중치↑** (예: 마감강도가 잘 맞으면 +8 → +12).
- 헛신호 많은 지표 → **가중치↓ 또는 제거**.
- false positive 많은 조건 → **필터 강화**(예: RSI 과매수 컷을 80→75).
- 국면 가점(`PHASE_ADJ`)을 **실제 국면별 적중률에 맞게 재조정**.
- 변경 후 반드시 **백테스트로 before/after 적중률 비교**(개선됐을 때만 채택).

## 3. ★ 자동 반복 튜닝 — `analyzer/tune.py` 신설 (핵심 개선)
사람이 매번 손대지 않고 **데이터가 가중치를 정하게** 한다.
- **입력**: `state/history/*` 예측 + 익일 실제(backtest와 동일 소스).
- **방법(택1, 단순→정교)**:
  1. **Lift 기반 가중치**: 각 feature의 (적중률_true − 적중률_false)에 비례해 가중치 산출(해석 쉬움).
  2. **로지스틱 회귀**: features → hit_up 학습, 계수를 가중치로(표준 라이브러리만으로 경사하강 구현 가능, pip 불필요).
- **출력**: `analyzer/state/weights.json`.
- **연동**: `run.py score()`가 `weights.json` 있으면 읽어 적용(없으면 현재 기본값). → 점진 학습.
- **자동화**: `scripts/install_cron.sh`에 주1회 라인 추가 (예: `0 8 * * 6  python3 analyzer/tune.py`).
- **⚠️ 과적합 방지(필수)**:
  - **Walk-forward**: 과거 구간으로 학습 → **최근 구간(out-of-sample)으로 검증** → OOS 적중률이 기존보다 **개선될 때만** weights.json 교체.
  - 표본 하한(예: 종목-예측 100건↑) 미달이면 튜닝 스킵.
  - 가중치 급변 클리핑(한 번에 ±X% 이내), 해석 가능 유지.

## 4. 피처 추가 (정확도 레버 — 데이터로 검증하며 1개씩)
- **시장 레짐 필터**: 코스피/코스닥 방향(약세장이면 후보·확신 축소). ← 효과 클 가능성.
- **수급**: 외국인·기관 순매수(네이버 제공) — 종가베팅 갭상승과 상관.
- **분봉 마감강도/체결 흐름**: 장 막판 강세 여부(이미 fchart 분봉 있음).
- **갭 통계**: 종목의 과거 오버나이트 갭 분포(갭상승 경향 종목).
- **섹터 강도**: 같은 섹터 동반 강세인지.
> 추가 시마다 백테스트로 **적중률 기여 검증** → 도움 되는 것만 유지.

## 5. 검증 체계 강화
- `backtest.py`에 **walk-forward / train·test 분리** 추가(과적합 모니터).
- **종목·국면·요일별** 적중률 세분 리포트.
- 사이트 `/forecast`에 적중률 추이(최근 N일) 노출 → 신뢰·자기점검.

## 6. 실행 순서(권장)
1. (2주 후) 0·1번: 데이터 충분한지 확인 + 진단.
2. 2번: 근거기반 수동 가중치 1차 조정 → 백테스트 before/after.
3. 3번: `tune.py`(자동 튜닝) + walk-forward 검증 → cron 등록.
4. 4번: 피처 1~2개 추가(레짐·수급 우선) → 검증.
5. 5번: 검증 체계·UI 강화.

## 정직 가드 (반드시 지킬 것)
- **표본 부족 시 튜닝 금지** — 노이즈 과적합이 정확도를 오히려 떨어뜨림.
- **OOS(검증셋) 개선 없으면 채택 금지** — in-sample 좋아보이는 건 착시.
- **적중률 상한은 현실적으로 60%대** — "시간 지나면 90%"는 없음. 손절·분산 전제.
- 갭 리스크(야간·미국장)는 통제 불가 — 모델 밖 변수.

> 관련 파일: `analyzer/run.py`(score·PHASE_ADJ) · `analyzer/backtest.py` · `analyzer/state/{history,backtest.json,calibration.json}` · `scripts/screener.py`(필터) · `scripts/install_cron.sh`(튜닝 cron). 전체 구조는 `analyzer/README.md`·`CLAUDE.md`.
