## 1. 결론

현재 저장된 동일 기준 표본에서는 `강한흔들기 AND 조합D`보다
`조합D 단독 + 수상함 점수 75점 이상`이 더 크고 재현성 있는 익일 고가 표본을 보유한다.

따라서 현행 `rank4-v2`의 최우선 순서를 다음처럼 바꾸는 것이 합리적이다.

```text
현재:
  bucket 0 = 강한흔들기 AND 조합D → 강한흔들기 Tier1 단독
  bucket 4 = 조합D 단독 + 75점 이상

수정:
  bucket 0 = 조합D 단독 + 75점 이상
  bucket 4 = 강한흔들기 AND 조합D → 강한흔들기 Tier1 단독(소표본 관찰)
```

중요한 제한:

- **모든 조합D 단독을 최우선으로 올리지 않는다.**
- `suspicion_score >= 75`를 함께 만족한 조합D 단독만 최우선이다.
- 조합D 단독이지만 75점 미만이면 기존 bucket 6을 유지한다.
- 강한흔들기와 조합D의 교집합을 조합D **단독**으로 잘못 포함하지 않는다.
- ranking만 바꾸며 스캐너 진입조건, 배지 정의, 자동매매 안전필터, 매수 종목 수, 매수 비중은 변경하지 않는다.

## 2. 수정이 필요한 원인

### 2.1 과거 용어 혼합

이전 분석에서 `조합D 단독`, `강한흔들기 OR 조합D`,
`강한흔들기 AND 조합D`가 서로 섞여 보고된 적이 있다.

현재 확정된 상호배타 정의는 다음과 같다.

```text
강한흔들기:
  shakeout = true
  very_good = true

조합D 단독:
  shakeout = true
  very_good = false
  strength_tier >= 3

강한흔들기 AND 조합D:
  shakeout = true
  very_good = true
  strength_tier >= 3

약한흔들기:
  shakeout = true
  very_good = false
  strength_tier < 3
```

`very_good`이 누락 또는 `None`이면 어느 단독·강약 분류에도 추정 편입하지 않고
`분류 미확인`으로 남긴다.

`very_good`은 `dd6 <= -30%`인 흔들기이고,
`strength_tier >= 3`은 2일 회전율 밴드와 60일 고점 대비 낙폭 밴드의 결합인 조합D다.
두 필드는 서로 다른 축이므로 OR와 AND를 바꿔 읽으면 안 된다.

### 2.2 현행 최상위 근거의 표본 부족

현행 `rank4-v2`는 `강한흔들기 AND 조합D`를 bucket 0 내부 최우선으로 올렸다.
그러나 2026-07-23 live final에서 새 bucket 4에 해당하는 표본은
`CS(강한흔들기 AND 조합D)`와 `금호전기(Tier1 단독)` 2건뿐이다.
`강한흔들기 AND 조합D` live 교집합 자체는 CS 1건이며, 서산은 백필 표본이므로
live와 합쳐 `2/2`라고 말하지 않는다. 이 소표본 적중률을 최우선 기대확률로 사용할 수 없다.

### 2.3 조합D 단독 + 75점의 표본 우위

신호 당시 저장된 필드를 현재의 상호배타 정의로 분류하고,
실제 final 카드만 사용하면 다음과 같다.

| 조건 | n | 고유종목 | 신호일 수 | 익일 +7 | 익일 +13 | 익일 -5 |
|---|---:|---:|---:|---:|---:|---:|
| 조합D 단독 전체 | 20 | 17 | 11 | 16/20, 80.0% | 14/20, 70.0% | 13/20, 65.0% |
| **조합D 단독 + 75점 이상** | **15** | **14** | **10** | **13/15, 86.7%** | **11/15, 73.3%** | **10/15, 66.7%** |
| 조합D 단독 + 80점 이상 | 11 | 11 | - | 10/11, 90.9% | 9/11, 81.8% | 7/11, 63.6% |
| 조합D 단독 + 75점 미만 | 5 | 5 | - | 3/5, 60.0% | 3/5, 60.0% | 3/5, 60.0% |

`조합D 단독 + 75점 이상` final 표본의 추가 수치는 다음과 같다.

- 평균 익일 최고수익률: `+17.59%`
- 중앙 익일 최고수익률: `+16.64%`
- 평균 종가수익률: `+4.15%` — 참고값일 뿐 순위 변경의 주 KPI로 사용하지 않는다.
- `+7`과 `-5` 동시 도달: `8/15`
- +7 적중률 Wilson 95% 구간: 약 `62.1~96.3%`

실제 `rank4-v1` EOD forward에서도 같은 조건은 다음과 같다.

| 모집단 | n | 익일 +7 | 익일 +13 | 익일 -5 |
|---|---:|---:|---:|---:|
| 조합D 단독 + 75점 이상, rank4-v1 EOD | 7 | 6/7, 85.7% | 5/7, 71.4% | 4/7, 57.1% |
| 그중 80점 이상 | 5 | 5/5, 100% | 4/5, 80.0% | 3/5, 60.0% |

즉 소급표본 하나만 보고 올리는 것이 아니라,
현재 final 카드와 저장된 EOD forward가 같은 방향을 보인다.

### 2.4 전수조사의 범위 제한

`data/shakeout_backfill.json`은 저장 당시 89종목 유니버스에서 70일을 재구성한 자료다.
전체 KRX/KOSDAQ 무편향 6개월 전수자료가 아니며 생존·유니버스 선택 편향이 있을 수 있다.

따라서 새 최우선 근거값은 다음 순서로 사용한다.

1. 신호 당시 필드가 저장된 실제 final 카드 `n=15`
2. 실제 rank4-v1 EOD forward `n=7`
3. 백필·소급자료는 방향 확인용 보조근거

백필과 live를 합친 `40/49 = 81.6%`를 순수 forward 승률이라고 쓰면 안 된다.

## 3. 순위 변경의 정확한 목표

### 3.1 최우선 조건

Python에서 다음 조건을 하나의 helper로 만든다.

```python
def _is_combo_d_only(x):
    return (
        x.get("shakeout") is True
        and x.get("very_good") is False
        and _is_combo_d(x)
    )


def _is_combo_d_only_75(x):
    score = _as_float(x.get("suspicion_score"), 0) or 0
    return _is_combo_d_only(x) and score >= 75
```

Fail-safe 원칙:

- `very_good=False`가 명시적으로 저장된 현재 신호만 최우선 승격한다.
- `very_good`이 누락되거나 `None`이면 “조합D 단독”으로 추정해 최상단에 올리지 않는다.
- `strength_tier`가 없거나 숫자로 변환되지 않으면 승격하지 않는다.
- 경계값은 `score >= 75.0`이다. `74.99`는 승격하지 않는다.

현재 웹 배지가 `very_good !== true`로 결측까지 조합D 단독처럼 보이게 한다면,
UI도 `very_good === false`와 일치시키고 결측은 `강도/분류 미확인`으로 표시한다.
현재 신규 스캔은 boolean을 저장하지만 구버전·손상 데이터의 오승격을 막기 위한 조치다.

### 3.2 권장 bucket 배치

bucket 번호 전체를 한 칸씩 미는 방식은 금지한다.

이유:

- bucket 1은 `급소+회전150` kill switch의 고정 입력이다.
- bucket 2·3은 저점매집 kill switch의 고정 입력이다.
- 웹은 bucket 1을 `폭발형`으로 별도 표시한다.
- 과거 signal-time bucket과 새 bucket의 의미는 모델 버전으로 분리해야 한다.
- 전체 번호 이동은 자동매매·UI·성과표 회귀 범위를 불필요하게 키운다.

최소 안전안은 **bucket 0과 bucket 4의 대상군만 교체**하는 것이다.

| bucket | 수정 후 의미 |
|---:|---|
| **0** | **조합D 단독 + 75점 이상** |
| 1 | 급소+회전150 — 변경 없음 |
| 2 | 저점매집+회전90 — 변경 없음 |
| 3 | 저점매집 기타 — 변경 없음 |
| **4** | 강한흔들기 AND 조합D → 강한흔들기 Tier1 단독, 소표본 관찰 |
| 5 | 조합D가 아닌 흔들기+75점 등 기존 조건 유지 |
| 6 | 조합D 단독+75점 미만 |
| 7~11 | 기존 의미 유지 |

`rank_bucket_info()`의 조건 순서는 다음처럼 한다.

```text
1. 강한흔들기 AND 조합D                          → bucket 4
2. 강한흔들기 Tier1 단독                         → bucket 4
3. 급소+회전150                                  → bucket 1
4. 저점매집+회전90                               → bucket 2
5. 저점매집 기타                                 → bucket 3
6. 조합D 단독 + score>=75                         → bucket 0
7. 기존 bucket 5~11 조건                         → 기존 유지
```

주의:

- 최종 정렬은 bucket 숫자가 작은 순서이므로 bucket 0은 최상단이다. 다만 분류 branch는 기존
  위치를 유지해 조합D 단독+75가 급소·저점매집과 중복되면 기존 bucket 1~3에 남긴다.
- `_is_very_good_combo_d()`는 `_is_combo_d_only_75()`와 상호배타여야 한다.
- 기존 `elif shakeout and comboD and score>=75 → bucket 4`는
  조합D 단독+75가 앞에서 bucket 0으로 소진된 후 강한흔들기를 잘못 재포획하지 않도록 정리한다.
- 조합D 단독 75점 미만은 계속 bucket 6이다.

### 3.3 bucket 0 내부 순서

bucket 0에는 조합D 단독+75점 이상만 들어가므로 별도 복잡한 tier 정렬을 추가하지 않는다.
기존 `rank_sort_key()`의 `-suspicion_score`를 이용하여:

```text
score 80 이상 → score 75~79
```

순으로 자연스럽게 정렬한다.

동점일 때는 기존 순서인:

1. fade
2. 폭발일 회전율
3. 2일 회전율
4. 종목코드

를 유지한다. 새로운 임의 가중치를 추가하지 않는다.

bucket 4 내부에서는 기존 `_very_good_sort_rank()`를 유지해:

1. 강한흔들기 AND 조합D Tier1
2. 강한흔들기 AND 조합D Tier2
3. 강한흔들기 Tier1 단독

순으로 관찰한다.

## 4. 모델 버전과 적용일

순위 의미가 바뀌므로 기존 `rank4-v2`를 그대로 사용하면 안 된다.

권장:

```text
RANK_POLICY_NAME = rank4
RANK_MODEL_VERSION = rank4-v3
```

적용일:

- 2026-07-24 첫 정규 레이더 실행 전에 배포가 끝나면 `effective_from=20260724`
- 장 시작 후 배포한다면 그날 중간부터 섞지 말고 **다음 실제 KRX 거래일**로 미룬다.
- 2026-07-23 history와 decision을 v3로 소급 재작성하지 않는다.

`scripts/radar_backtest.py`의 모델별 forward 하한에는 최소 다음을 명시적으로 보존한다.

```python
KNOWN_FORWARD_EFFECTIVE_FROM = {
    "rank4-v1": "20260713",
    "rank4-v2": "20260724",
    "rank4-v3": "<실제 적용 거래일>",
}
```

v2가 실제 신호를 한 건도 만들기 전에 v3로 대체되더라도 삭제하거나 v1에 합치지 않는다.
모델별 표본은 그대로 `n=0`일 수 있다.

`RANK_MODEL_SOURCE_COMMIT`에는 `working-tree` 같은 임시 문자열을 남기지 않는다.
정책 구현 커밋의 승인 기준점/부모 커밋을 저장하고 `radar_json_store.DEFAULT_MODEL`과 정확히 일치시킨다.

## 5. 통계 스냅샷 수정

`RANK_BUCKET_BASELINES`의 bucket 0을 현재 live final 근거로 갱신한다.

권장 고정값:

```text
label: 조합D 단독+75점 최우선
n: 15
unique_n: 14
touch7_rate: 86.7
expected_high_pct: 17.59
avg_return: 4.15
note:
  live final 13/15 +7, 11/15 +13,
  rank4-v1 EOD 6/7 +7. 백필과 forward 혼합 금지.
```

bucket 4는 새 의미에 맞게 다시 계산한다.
현재 live final에서 `강한흔들기 AND 조합D 또는 Tier1 단독`은 2건뿐이므로,
`2/2=100%`를 기대확률처럼 노출하지 않는다.

권장 처리:

- `strength = observe`
- `표본주의` 명시
- live `n=2`, 고유종목 2개 공개
- 기대확률은 `None`으로 두거나, 별도 소급 prior를 사용한다면 출처·소급임을 note에 명시
- signal-time prior, retro, forward를 절대 한 숫자로 합치지 않는다.

`RANK_BUCKET_PRIORS`와 `radar_json_store.DEFAULT_MODEL.prior`도 새 근거로 갱신한다.

예:

```text
source: live_final_and_rank4_v1_eod_20260723
strength: medium
summary: 조합D 단독+75점 final 13/15, rank4-v1 EOD 6/7
```

`80점 이상 10/11`은 bucket 0 내부 정렬 근거이며 별도 하드 bucket을 새로 만들지 않는다.
현재 표본에서 80점을 새로운 영구 임계값으로 과최적화하지 않는다.
