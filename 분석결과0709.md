## 핵심 기준

- 반드시 **익일 고가 기준**만 사용했다.
- 사용 지표: `next_high_pct = (익일 고가 / 신호일 종가 - 1) * 100`
- 종가 수익률(`return_pct`)이나 익일 종가 적중률은 이 문서의 조합 우선순위에 쓰지 않았다.
- 평가 완료 표본: **156건**
- 평가 완료 신호일: **20일**
- 평가 완료 범위: `2026-06-10` ~ `2026-07-07`
- `2026-07-08`, `2026-07-09` 신호는 아직 익일 고가가 없어 제외했다.

## 용어

- `+7 터치`: 익일 고가가 신호일 종가 대비 +7% 이상 도달한 비율
- `+11 터치`: 익일 고가가 +11% 이상 도달한 비율
- `+13 터치`: 익일 고가가 +13% 이상 도달한 비율
- `+20 터치`: 익일 고가가 +20% 이상 도달한 비율
- `매우좋음`: 흔들기 중 `dd6 <= -30`
- `매우좋음후보`: 흔들기 중 `-30 < dd6 <= -25`
- `peakTurnover`: 폭발일 거래량 / 유통주식수 회전율
- `peakDD`: 고점 대비 낙폭

## 실전 우선순위 제안

현재 데이터만 기준으로 하면 다음 순서가 합리적이다.

1. **저점매집 + `peakTurnover >= 90`**
   - n=13, 평균 익일고가 18.17%, +7 터치 84.6%
   - 표본과 성과의 균형이 가장 좋다.

2. **매수급소 + `peakTurnover >= 90`**
   - n=5, 평균 익일고가 22.12%, +7/+11 터치 100%
   - 표본은 적지만 매우 강하다.

3. **흔들기 + `score >= 75` + `peakDD <= -30`**
   - n=3, 평균 익일고가 25.24%, +13 터치 100%
   - 강하지만 표본이 작다.

4. **매우좋음**
   - n=2, 평균 익일고가 29.97%, 전부 상한가급
   - 가장 강하게 관찰되지만 아직 표본이 너무 작다.

5. **저점매집 전체 / 매수급소 전체 / 흔들기 전체**
   - 필터가 없을 때도 전체 평균보다 좋다.
   - 다만 최우선 매수 후보는 위의 `peakTurnover`나 `score/peakDD` 필터를 붙인 조합이 낫다.

6. **매우좋음후보**
   - 현재 기준 단독 우선순위 낮음.
   - 후보 단계에서는 관찰만 하고, `매우좋음`으로 승격되거나 다른 강한 신호와 중첩될 때만 의미를 둔다.

## 재료 등급 분석 상태

재료 등급 필드는 저장되기 시작했다. 그러나 아직 익일 고가 기준으로 평가 완료된 표본은 없다.

현재 저장된 재료 등급 표본:

|등급|저장 표본|익일고가 평가 완료|
|---|---:|---:|
|S|5|0|
|A|2|0|
|B|1|0|
|C|3|0|
|D|2|0|
|N|5|0|

해석:
- 재료 S/A/B/C/D/N은 아직 백테스트 결론을 내릴 수 없다.
- `S/A + 매수급소`, `S/A + 저점매집`, `S/A + 흔들기`, `S/A + 매우좋음` 조합은 다음 평가 사이클 이후 별도 검증해야 한다.
- 현재 문서의 조합 우선순위에는 재료 등급을 반영하지 않았다.

## 현재 결론

가장 실전적인 조합은 **저점매집 + 폭발일 회전율 90% 이상**이다.  
가장 폭발적인 관찰 조합은 **매수급소/저점매집 + 폭발일 회전율 150% 이상**, 그리고 **매우좋음**이다.  
하지만 n=2~4 조합은 아직 과최적화 가능성이 크므로, 자동 룰로 확정하지 말고 강한 관찰/알림 우선순위로만 둔다.

## 추후 업데이트 체크리스트

- 평가 완료 표본이 늘어나면 이 문서를 갱신한다.
- `매우좋음`은 n>=10 전까지 확정 룰 금지.
- `매수급소 + peakTurnover >= 150`은 n>=10 전까지 확정 룰 금지.
- `저점매집 + peakTurnover >= 90`은 현재 가장 먼저 실전 적용 검토 가능.
- 재료 등급은 평가 완료 표본이 생긴 뒤 반드시 별도 표로 추가한다.

## 재계산용 참고 스크립트

```bash
cd /Users/jinjin/kiwoomnews
python3 - <<'PY'
import json, pathlib, statistics
root=pathlib.Path('/Users/jinjin/kiwoomnews')
rows=[]
for p in sorted((root/'data/radar_history').glob('*.json')):
    d=json.load(open(p,encoding='utf-8'))
    date=d.get('date') or p.stem
    for code,s in (d.get('suspects') or {}).items():
        r=s.get('result') or {}
        if not s.get('evaluated') or not r or r.get('next_high_pct') is None:
            continue
        rows.append({
            'date':date,
            'code':code,
            'name':s.get('name'),
            'next_high_pct':float(r['next_high_pct']),
            'very_good':bool(s.get('very_good')),
            'very_good_candidate':bool(s.get('very_good_candidate')),
            'shakeout':bool(s.get('shakeout')),
            'geupso':bool(s.get('geupso')),
            'low_accum':bool(s.get('low_accum')),
            'peak_turnover_pct':s.get('peak_turnover_pct'),
            'peak_dd_pct':s.get('peak_dd_pct'),
            'score':s.get('suspicion_score'),
            'rank':s.get('rank'),
        })

def stat(name, pred):
    g=[x for x in rows if pred(x)]
    vals=[x['next_high_pct'] for x in g]
    if not vals:
        print(name, 'n=0')
        return
    def rate(t):
        return round(sum(v>=t for v in vals)/len(vals)*100,1)
    print(name, {
        'n':len(vals),
        'avg':round(sum(vals)/len(vals),2),
        'median':round(statistics.median(vals),2),
        '+7':rate(7),
        '+11':rate(11),
        '+13':rate(13),
        '+20':rate(20),
    })

stat('매우좋음', lambda x:x['very_good'])
stat('매우좋음후보', lambda x:x['very_good_candidate'])
stat('흔들기', lambda x:x['shakeout'])
stat('흔들기 score>=75 peakDD<=-30',
     lambda x:x['shakeout'] and (x['score'] or 0)>=75 and x['peak_dd_pct'] is not None and x['peak_dd_pct']<=-30)
stat('매수급소', lambda x:x['geupso'])
stat('매수급소 peakTurnover>=90',
     lambda x:x['geupso'] and (x['peak_turnover_pct'] or -999)>=90)
stat('매수급소 peakTurnover>=150',
     lambda x:x['geupso'] and (x['peak_turnover_pct'] or -999)>=150)
stat('저점매집', lambda x:x['low_accum'])
stat('저점매집 peakTurnover>=90',
     lambda x:x['low_accum'] and (x['peak_turnover_pct'] or -999)>=90)
stat('저점매집 rank<=10 peakTurnover>=150',
     lambda x:x['low_accum'] and (x['rank'] or 999)<=10 and (x['peak_turnover_pct'] or -999)>=150)
PY
```
