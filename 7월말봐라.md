## 2. 회장님이 지시할 문장

> **"7월말봐라.md대로 rank4-v1 forward를 KRX·NXT·EOD별로 채점하라. prior와 retro는 참고로 분리하고, n=1도 빠뜨리지 말고 순위 변경 전 내 승인을 받아라."**

### Claude 실행 체크리스트 (Claude용 지침)

1. **원본 검증** — manifest와 decision/evaluation JSON checksum을 확인하고 7/10 mixed-deployment, 모델 없는 legacy, bucket 없는 표본이 forward에서 제외됐는지 확인한다.
2. **모집단별 버킷 채점** — KRX 15:18, NXT 19:50, operational EOD를 따로 표로 만든다. n·고유종목·+7/+13·Wilson·평균/중앙/최저 고가·종가평균을 모두 제시한다. 기존 `final`은 별도 하위호환 표다.
3. **상대순위 채점** — 모집단별 Top1 winner, Top3 안 winner, Spearman, NDCG를 보고한다. 단일후보 날을 Top1 분모에 섞지 않는다. 동률 winner 처리도 명시한다.
4. **킬스위치·승격 판정** — 사전 고정 기준 그대로 적용하되 forward EOD만 actionable이다. 소표본 결과도 보고하되 자동 재정렬하지 않는다. **기준 사후 조정 금지.**
5. **75 vs 80·상단컷·재료·경보 가설** — 기존 가설은 forward에서 별도 코호트로 비교한다. 표본이 작다는 이유로 행을 삭제하지 말고 n과 종목명을 같이 제시한다.
6. **반드시 지킬 것** — 자동튜닝 금지. prior를 실측 한 번으로 삭제하지 않는다. 변경 제안은 before/after suspects 순서와 자동매매 top3 diff를 먼저 제시하고 회장님 승인 후 적용한다. 비중·수량·안전필터는 별도 합의 전 불변이다.
