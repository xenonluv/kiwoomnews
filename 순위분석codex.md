작성일: 2026-07-10  
대상 데이터: `data/radar_history/*.json` 중 `evaluated=true`이고 `result.next_high_pct`가 있는 표본  
평가 표본: 179건  
평가 범위: 2026-06-10 ~ 2026-07-09  
목표 지표: 익일 장중 고가 +7% 이상 터치율과 평균 익일고가

## 핵심 결론

현재까지는 suspects 전체를 그대로 보면 +7% 터치율이 48.6%에 그친다. 그러나 강한 조건이 붙은 후보만 보면 +7% 터치율이 75~88%대로 올라간다. 따라서 1~10위 정렬은 단순 `shakeout`, `very_good_candidate`, `fade_pct`, `suspicion_score` 순서가 아니라 **익일 +7% 터치율이 높은 조합을 우선하는 계층형 정렬**로 바꾸는 것이 맞다.

특히 이번 분석에서 중요한 사실은 다음과 같다.

- `매수급소` 단독은 생각보다 약하다. n=12, +7% 터치율 66.7%, 평균 익일고가 +15.35%.
- `매수급소 + peakTurnover >= 150`만 강하다. n=5, +7/+13/+20% 터치 전부 100%, 평균 익일고가 +27.42%.
- `저점매집 + peakTurnover >= 90`은 표본과 승률 균형이 가장 좋다. n=15, +7% 터치율 86.7%, 평균 익일고가 +17.00%.
- `경고지정`은 위험하지만 고가 터치 통계는 강하다. n=16, +7% 터치율 87.5%, 평균 익일고가 +18.50%.
- `흔들기 + 점수75+`는 흔들기 내부의 실전 필터로 쓸 수 있다. n=12, +7% 터치율 83.3%, 평균 익일고가 +17.39%.
- `매우좋음`은 n=2라 폭발력은 최강이지만 아직 확정 룰로는 표본이 부족하다.
- `매우좋음후보`는 이름과 달리 약하다. n=6, +7% 터치율 66.7%, 평균 익일고가 +11.44%라 상위 정렬에서 내려야 한다.

## 현재 정렬의 문제

현재 `scripts/radar.py`의 정렬은 대략 다음 우선순위다.

1. 매우좋음
2. 매우좋음후보
3. 흔들기
4. 경고 해제 예정 / 투자위험 해제 직후
5. 매수급소
6. 흔들기 내부 `fade_pct`, `suspicion_score`, `dd6_pct`
7. 최종 `suspicion_score`

문제는 다음과 같다.

- `매우좋음후보`가 일반 흔들기보다 위인데, 실제 성과는 좋지 않다.
- `매수급소`는 단독으로 상위 승격하기에는 약하고, `peakTurnover >= 150`이 붙을 때만 최상위급이다.
- `저점매집 + peakTurnover >= 90`은 통계가 강한데 현재 정렬 승격 축에서 빠져 있다.
- `경고지정`은 리스크 때문에 자동매수와 별도 판단이 필요하지만, 익일 고가 터치 예측 관점에서는 상위 후보로 볼 근거가 있다.
- `점수75+`, `흔들기+점수75+`, `흔들기+조합D` 같은 실측 강한 필터가 현재 정렬의 명시 계층이 아니다.

## 새 순위 재정렬 제안

정렬 목표는 두 가지를 분리해야 한다.

- **고가 터치 예측 순위**: 익일 +7% 이상 고가를 찍을 가능성이 높은 순서.
- **자동매수/실전 안정 순위**: 고가 터치뿐 아니라 리스크, 표본 수, 하방 실패를 같이 본 순서.

현재 회장님이 보는 suspects 1~10위는 매수 판단에 쓰이므로, 기본 정렬은 **실전 안정 순위**를 쓰고, 카드에는 `고가터치형`, `폭발형`, `위험고탄력` 배지를 붙이는 방식이 좋다.

## 새 정렬 로직 초안

코드 적용 시에는 기존 boolean 정렬키보다 `rank_bucket` 숫자를 먼저 부여하는 방식이 명확하다.

```text
rank_bucket 0: 매수급소 + peakTurnover>=150
rank_bucket 1: 매우좋음 Tier1
rank_bucket 2: 저점매집 + peakTurnover>=90
rank_bucket 3: 경고지정 + 점수70+ 또는 alert_release/alert_risk_released
rank_bucket 4: 흔들기 + 조합D + 점수75+
rank_bucket 5: 흔들기 + 점수75+
rank_bucket 6: 저점매집 전체
rank_bucket 7: 흔들기 + 조합D
rank_bucket 8: 점수75+ 전체
rank_bucket 9: 흔들기 전체
rank_bucket 10: 기타 suspects
rank_bucket 20: 매우좋음후보 단독
rank_bucket 30: 매수급소 단독 또는 peakTurnover 90~150
```

같은 bucket 안에서는 다음 순서로 세부 정렬한다.

1. `suspicion_score` 높은 순
2. `next_high_expected_pct` 높은 순(별도 산출 시)
3. 흔들기면 `fade_pct` 높은 순
4. `peak_turnover_pct` 또는 `turnover_2d_pct` 높은 순
5. `run_6d_pct`가 30% 이상이면 상단 기대를 낮추고 같은 bucket 내부 후순위

## 화면 노출 제안

순위만 바꾸면 왜 위에 있는지 이해하기 어렵다. 카드에 다음 배지를 추가하면 판단이 쉬워진다.

- `+7 통계강`: 해당 조합 +7% 터치율 80% 이상
- `폭발형`: 평균 익일고가 20% 이상 또는 +20% 터치율 50% 이상
- `표본주의`: n<10
- `고위험`: 경고/위험 지정, 거래정지 가능성
- `상단컷`: run_6d 30% 이상, 재료 약함, 고회전 소진

예시:

```text
서산: 흔들기 + 조합D + 점수80+ + 경고지정
배지: +7 통계강 / 고위험 / 상단컷
정렬: bucket 3~5 사이. 고가 터치는 강하지만 과열 때문에 예상 상단은 낮춤.
```

## 전진검증 방식

순위 재정렬은 한 번에 코드 확정하지 말고 전진검증으로 적용해야 한다.

1. 매일 장마감 직후 suspects마다 `rank_bucket`, `rank_reason`, `expected_high_pct`, `expected_touch7_rate`를 history에 저장한다.
2. 익일 평가 시 `next_high_pct`와 비교한다.
3. bucket별로 `n`, 평균 고가, 중앙값, +7/+13/+20 터치율을 누적한다.
4. n<10 bucket은 `표본주의`로 표시하고 정렬 가중치를 과하게 주지 않는다.
5. n>=20이 된 bucket만 기본 정렬 공식으로 승격한다.

## 현재 기준 최종 판단

지금 바로 순위 기준을 바꾼다면 다음 원칙이 가장 중요하다.

- **1위권은 매수급소 전체가 아니라 `매수급소 + peakTurnover>=150`만 올린다.**
- **안정 1순위는 `저점매집 + peakTurnover>=90`이다.**
- **흔들기는 `점수75+` 또는 `조합D+점수75+`가 붙을 때 상단으로 올린다.**
- **경고지정은 고가 터치 통계가 강하지만 리스크 배지를 반드시 붙이고 자동비중 확대는 금지한다.**
- **매우좋음후보는 이름과 달리 상위 정렬하면 안 된다.**

이 기준이면 suspects 1~10위가 단순 수상함 점수 순서가 아니라, 실제 익일 고가 +7% 이상을 찍을 확률이 높은 순서로 재배치된다.
